## CRITICAL DATA EXTRACTION RULES

### ⚠️ FIRST: CLASSIFY DOCUMENTS AS OLD vs CURRENT

**Before extracting any data**, classify each uploaded document:

| Document Type | Classification | Use For |
|--------------|----------------|---------|
| Visa Worksheet | **CURRENT** | Part 4 (Proposed US Employment) - address, wages, job title |
| Current Employment Letter | **CURRENT** | Part 4 duties, compensation |
| Prior I-129S (old form) | **OLD** | Part 5 (Foreign Employment History) - past positions, dates |
| Old Employment Letters | **OLD** | Part 5 historical work information |
| Previous Visa Applications | **OLD** | Part 5 historical dates |

**CRITICAL RULES:**
- **Part 4 (Proposed US Employment)**: ALWAYS use **CURRENT** documents (Visa Worksheet). NEVER copy address, wages, or job info from prior I-129S - that data is outdated.
- **Part 5 (Foreign Employment History)**: Use **OLD** documents (prior I-129S, old records) for historical dates and positions.

---

### 1. PHYSICAL ADDRESS vs. MAILING ADDRESS
- **Rule**: If the Petitioner's Mailing Address (Part 1, Line 2) is a P.O. Box, you **MUST** search the documents (especially the "Visa Worksheet" or "U.S. Contact Information" sections) for a **Physical Address**.
- **Action**: Fill Part 1, Line 4 (Physical Address) with this location.
- **Context**: USCIS requires a physical location for the petitioner. A P.O. Box is NOT sufficient for the physical address fields.

### 2. EMPLOYEE COUNT & "50+ EMPLOYEES" CHECKBOX
- **Rule**: To answer Part 1, Line 9 ("Does the petitioner employ 50 or more individuals?"), you must infer the answer if not explicitly stated.
- **Action**: Check the Support Letter or Company Overview.
    - If it says "employs approximately 43 employees", check **NO**.
    - If it says "employs over 100 people", check **YES**.
- **Context**: This is a critical eligibility question. Do not leave it blank if the data exists in the text.

### 3. EMAIL ADDRESSES
- **Rule**: If a specific "Petitioner Email" is not found in the form headers, use the **Signer's Email** found in the Support Letter signature block.
- **Action**: Fill Part 1, Line 7 with the signer's email.

## CRITICAL FORMATTING RULES

### Standard Text Formatting

- **ALL TEXT FIELDS**: UPPERCASE for names and addresses
- **Phone numbers**: MUST use dashes (e.g., "310-227-4403") or +Code format
- **Dates**: MM/DD/YYYY format
- **Email addresses**: UPPERCASE
- **States**: 2-letter codes (e.g., "CA", "NY")
- **Checkboxes**: "Y" if checked, omit if unchecked

### Unit Checkboxes (APT/STE/FLR) - CRITICAL FORMAT

For address unit types (Apartment, Suite, Floor), use ONE field WITHOUT index, with value EXACTLY "APT", "STE", or "FLR":

**CORRECT:**
```json
"form1[0].#subform[0].Pt1Line2c_Unit": "STE"
```

**WRONG:**
```json
"form1[0].#subform[0].Pt1Line2c_Unit[0]": "1"
"form1[0].#subform[0].Pt1Line2c_Unit[1]": "STE"
```

Unit checkbox fields (use base name WITHOUT index):
- `Pt1Line2c_Unit` → Petitioner mailing address unit type
- `Pt1Line4b_Unit` → Petitioner physical address unit type
- `Pt3Line6c_Unit` → Beneficiary foreign mailing address unit type
- `Pt3Line8b_Unit` → Beneficiary foreign physical address unit type
- `Pt4Line3b_Unit` → Proposed employment address unit type
- `Pt5Line3b_Unit` → Foreign employer address unit type

### Single-Line Field Character Limits - CRITICAL

To prevent text overflow past field boundaries, observe these character limits:

**Long fields (48 chars max):**
- Job titles: Abbreviate if needed (e.g., "SR. MGR TO DIV CONTROLLER")
- Business/Organization names
- Street addresses (use abbreviations: ST, AVE, BLVD, STE)
- Countries, provinces

**Medium fields (36 chars max):**
- Family name, Given name, Middle name

**Short fields (35 chars max):**
- City names
- Some address fields

**Examples of intelligent abbreviation:**
- "SPECIAL APPOINTED MANAGER TO DIVISION CONTROLLER" → "SR. APPOINTED MGR TO DIV CTRL" (29 chars)
- "BUNKYO GARDEN GATE TOWER, 18-1 KOISHIKAWA" → "BUNKYO GARDEN GATE TWR 18-1" (27 chars)
- "ASSISTANT VICE PRESIDENT OF OPERATIONS" → "ASST VP OF OPERATIONS" (21 chars)

If text cannot fit within the limit, prioritize the most identifying information and use standard abbreviations.
